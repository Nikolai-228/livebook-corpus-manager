# Модуль постобработки текстов

Модуль для очистки, нормализации и улучшения качества текстовых данных, извлечённых из документов (особенно PDF), а также очистки названий папок. Входит в состав проекта **«Живая книга ИМИ — Корпусный менеджер»**.

---

## 📋 Назначение

- Очистка текста от мусора (URL, номера страниц, сноски)
- Склейка разорванных слов в PDF-документах
- Нормализация заголовков документов
- Очистка названий папок
- Экспорт и импорт PDF-документов для внешней обработки
- Тематическое моделирование с использованием BERTopic

---

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install psycopg2-binary tqdm bertopic sentence-transformers umap-learn hdbscan scikit-learn pandas numpy
```

### 2. Настройка БД

В `config.py` укажи строку подключения:

```python
DATABASE_DSN = "postgresql://postgres:password@localhost:5432/livebook_corpus"
```

### 3. Запуск

```bash
# Обработать все документы и папки
python text_cleaner.py

# Обработать только документы
python text_cleaner.py --documents

# Обработать только папки
python text_cleaner.py --folders

# Пробный запуск без сохранения
python text_cleaner.py --dry-run

# Ограничить количество обрабатываемых записей
python text_cleaner.py --limit 100
```

## 📄 Обработка текста

### Очистка контента (`content`)

| Тип документа | Действие |
|---------------|----------|
| **Все** | Удаление URL, сносок `[1]`, `(1)`, номеров страниц |
| **PDF** | Склейка слов, разорванных дефисом и переносом строки |
| **Все** | Нормализация пробелов |

### Очистка заголовков (`title`)

| Правило | Пример |
|---------|--------|
| Удаление чисел в начале | `"42  Текст"` → `"Текст"` |
| Первая буква заглавная | `"текст"` → `"Текст"` |
| Если контент короткий (`< 50` символов) | `"Фото"` → `"Документ с фотографиями \"Фото\""` |

### Очистка названий папок (`folders.name`)

| Правило | Пример |
|---------|--------|
| Удаление чисел в начале | `"1.2 Папка"` → `"Папка"` |
| Первая буква заглавная | `"папка"` → `"Папка"` |
| Удаление лишних пробелов | `"Папка   с   пробелами"` → `"Папка с пробелами"` |

## 🔧 Экспорт и импорт

### Экспорт PDF в CSV

```bash
python export_pdfs_to_csv.py
```

**Аргументы:**

| Аргумент | Описание |
|----------|----------|
| `--output`, `-o` | Путь для сохранения CSV |
| `--limit`, `-l` | Ограничить количество PDF |
| `--no-content` | Не включать поле `content` (очень большое) |

### Импорт из CSV

```bash
python import_csv_to_db.py [путь_к_csv]
```

**Аргументы:**

| Аргумент | Описание |
|----------|----------|
| `csv_file` | Путь к CSV файлу (по умолчанию: `exports/pdf_deep.csv`) |
| `--dry-run` | Пробный запуск без сохранения |
| `--no-content` | Не обновлять поле `content` |
| `--update-title` | Обновлять поле `title` |

## 🧠 Тематическое моделирование (BERTopic)

Модуль использует **BERTopic** для автоматического распределения документов по темам на основе их содержания.

### Особенности

| Параметр | Значение |
|----------|----------|
| **Модель эмбеддингов** | `paraphrase-multilingual-mpnet-base-v2` (поддерживает русский язык) |
| **Снижение размерности** | UMAP (5 компонентов для работы на CPU) |
| **Кластеризация** | HDBSCAN |
| **Векторизация** | CountVectorizer с n-граммами (1-2) |

### Запуск

```bash
python topic_modeling.py
```

**Аргументы:**

| Аргумент | Описание |
|----------|----------|
| `--test`, `-t` | Тестовый запуск (первые 100 документов) |
| `--limit`, `-l` | Ограничить количество документов |
| `--no-save`, `-n` | Не сохранять результаты в БД |
| `--topics`, `-k` | Количество тем (по умолчанию: 20) |

### Сохранение результатов

Результаты сохраняются в БД:

1. **`documents.topic_id`** — ID темы для каждого документа
2. **`topics_summary`** — таблица с описаниями тем:

| Поле | Описание |
|------|----------|
| `id` | Первичный ключ |
| `topic_id` | ID темы (начинается с 1) |
| `name` | Название темы |
| `keywords` | Ключевые слова темы |
| `doc_count` | Количество документов в теме |
| `created_at` | Дата создания |

### Требования к данным

Для работы тематического моделирования необходима таблица **`documents_lemmatized`** с лемматизированными текстами. Если её нет — сначала запусти модуль лемматизации.
