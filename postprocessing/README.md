```markdown
# Модуль постобработки текстов

Модуль для очистки, нормализации и улучшения качества текстовых данных, извлечённых из документов (особенно PDF). Входит в состав проекта **«Живая книга ИМИ — Корпусный менеджер»**.

## 📋 Назначение

Модуль обеспечивает:
- Очистку текста от мусора (URL, ссылки, номера страниц, сноски)
- Склейку разорванных слов в PDF-документах
- Нормализацию заголовков (удаление цифр в начале, приведение к верхнему регистру)
- Автоматическое определение документов, содержащих только изображения
- **Экспорт и импорт PDF-документов** для внешней обработки

## 📁 Структура модуля

```
postprocessing/

├── __init__.py              # Инициализация модуля

├── config.py                # Конфигурация (БД, параметры обработки)

├── text_cleaner.py          # Основная очистка текста и заголовков

├── export_pdfs_to_csv.py    # Экспорт PDF документов в CSV

├── import_csv_to_db.py      # Импорт обработанных данных из CSV

```

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install psycopg2-binary tqdm
```

### 2. Настройка конфигурации

Отредактируйте `config.py`:

```python
# PostgreSQL
DATABASE_DSN = "postgresql://user:password@localhost:5432/your_db"
```

### 3. Запуск постобработки

```bash
# Обработать все документы
python text_cleaner.py

# Обработать только первые 100 документов (тестовый запуск)
python text_cleaner.py --limit 100

# Пробный запуск без сохранения в БД
python text_cleaner.py --dry-run
```

## 🔧 Основные компоненты

### `text_cleaner.py` — Основная очистка

| Функция | Назначение |
|---------|-------------|
| `clean_text_basic(text)` | Удаляет URL, сноски, номера страниц, лишние пробелы |
| `fix_pdf_text(text)` | Склеивает разорванные слова в PDF (например, "конеч-\nно" → "конечно") |
| `postprocess_text(text, doc_type)` | Комбинированная очистка с учётом типа документа |
| `clean_title(title, content)` | Очищает заголовок, добавляет префикс для документов без текста |
| `process_all_documents(limit, dry_run)` | Массовая обработка всех документов в БД |

**Пример работы `clean_title`:**
```python
# Было: "1.2.3 Воспоминания о стройотряде"
# Стало: "Воспоминания о стройотряде"

# Если content пустой или очень короткий:
# Было: "IMG_123.jpg"
# Стало: "Документ с фотографиями \"IMG_123.jpg\""
```

### Очистка текста

| Тип мусора | Пример | Результат |
|------------|--------|-----------|
| URL | `https://drive.google.com/...` | удаляется |
| Сноски | `[1]`, `(2)`, `{*}` | удаляются |
| Номера страниц | `Страница 15`, `- 4 -` | удаляются |
| Разорванные слова | `конеч-\nно` | `конечно` |

## 📤 Экспорт PDF для внешней обработки

Если стандартная очистка не даёт желаемого результата, вы можете экспортировать PDF-документы, обработать их внешней системой (например, нейросетью, другим OCR, ручной проверкой), а затем импортировать обратно.

### Экспорт PDF в CSV

```bash
# Экспорт всех PDF документов (включая поле content)
python export_pdfs_to_csv.py

# Экспорт без поля content (если нужно только метаданные)
python export_pdfs_to_csv.py --no-content

# Экспорт только первых 50 PDF (для теста)
python export_pdfs_to_csv.py --limit 50

# Сохранить в указанный файл
python export_pdfs_to_csv.py --output /path/to/my_export.csv
```

**Что экспортируется:**
- Все поля таблицы `documents` (id, title, content, type, url, creation_date и др.)
- Только документы с `type = 'pdf'`
- Формат: CSV с разделителем `;` (точка с запятой)
- Кодировка: UTF-8 with BOM (совместима с Excel)

**Пример вывода:**
```

======================================================================
✅ Подключение к PostgreSQL установлено

   📄 Найдено PDF: 156

   💾 Путь: exports/pdf_export_20250612_153042.csv

✅ ЭКСПОРТ ЗАВЕРШЁН!
   📄 Документов: 156

```

## 📥 Импорт обработанных данных

После того как вы обработали PDF внешней системой (например, нейросетью Llama 3.2, ручной вычиткой, другим OCR), импортируйте изменения обратно в БД.

### Импорт из CSV

```bash
# Базовый импорт (обновляет только поле content)
python import_csv_to_db.py exports/pdf_export_20250612_153042.csv

# Импорт с обновлением заголовков
python import_csv_to_db.py exports/pdf_export_20250612_153042.csv --update-title

# Пробный запуск (только показать, что будет обновлено)
python import_csv_to_db.py exports/pdf_export_20250612_153042.csv --dry-run

# Импорт без обновления content (только title)
python import_csv_to_db.py exports/pdf_export_20250612_153042.csv --no-content --update-title
```

**Важные требования к CSV для импорта:**
- Файл должен содержать колонку `id` (идентификатор документа)
- Для обновления текста нужна колонка `content`
- Разделитель: `;` (точка с запятой)
- Кодировка: UTF-8

**Пример вывода:**
```

======================================================================

## 🔄 Типичный сценарий внешней обработки

```bash
# 1. Экспортируем PDF документы
python export_pdfs_to_csv.py --output ./pdf_export.csv

# 2. Обрабатываем CSV внешней системой
# Например, запускаем нейросеть для улучшения текста
python ../external_ai/process_pdfs.py --input ./pdf_export.csv --output ./pdf_processed.csv

# 3. Проверяем результат (опционально)
head -n 5 ./pdf_processed.csv

# 4. Импортируем обработанные данные (пробный запуск)
python import_csv_to_db.py ./pdf_processed.csv --dry-run

# 5. Реальный импорт
python import_csv_to_db.py ./pdf_processed.csv

# 6. Запускаем финальную постобработку
python text_cleaner.py
```

## ⚙️ Параметры обработки (config.py)

| Параметр | Значение | Назначение |
|----------|----------|------------|
| `BATCH_SIZE` | 50 | Количество документов между коммитами в БД |
| `MIN_TEXT_LENGTH` | 20 | Минимальная длина текста для считания его "содержательным" |
| `MAX_TEXT_LENGTH` | 100000 | Максимальная длина текста (отсечение очень больших документов) |
| `LOG_LEVEL` | "INFO" | Уровень детализации логов |

## 🐛 Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Ошибка подключения к БД | Проверьте `DATABASE_DSN` в `config.py` |
| CSV не импортируется | Убедитесь, что разделитель `;` и кодировка UTF-8 |
| Документы не найдены при импорте | Проверьте, что колонка `id` содержит корректные значения |
| Очень медленный импорт | Увеличьте `BATCH_SIZE` в `config.py` |
| Текст не очищается должным образом | Используйте экспорт/импорт для внешней обработки |

## 🔌 Интеграция с внешними системами

Модуль спроектирован для простой интеграции:

1. **Экспорт** → стандартный CSV с разделителем `;`
2. **Внешняя обработка** → любая система, умеющая читать/писать CSV
3. **Импорт** → обновление только изменённых полей

Это позволяет использовать:
- Нейросетевые модели (Llama 3.2, GPT, Yandex GPT)
- Специализированные OCR-системы (Tesseract, ABBYY)
- Ручную вычитку в Excel/Google Sheets
- Другие языковые процессоры (spaCy, NLTK, Natasha)
