# Модуль лингвистического анализа

Модуль для лингвистического анализа текстовых документов из базы данных PostgreSQL. Входит в состав проекта **«Живая книга ИМИ — Корпусный менеджер»**.

---

## 📋 Назначение

Модуль обеспечивает:
- Извлечение и анализ униграмм (отдельных слов) с подсчетом частоты и контекстов
- Поиск и анализ биграмм (пар слов) по частоте встречаемости
- Поиск и анализ триграмм (троек слов) по частоте встречаемости
- Извлечение коллокаций по метрике Likelihood Ratio
- Извлечение ключевых слов с использованием TF*ICTF
- Поиск по регулярным выражениям в оригинальных и лемматизированных текстах
- Извлечение именованных сущностей (люди, организации, места, даты, деньги, проценты)

## 🔧 Инструменты и их использование

### 1. Униграммы (`unigrams_search.py`)

Поиск и анализ отдельных слов с подсчетом частоты и контекстов.

**Режимы работы:**
1. **Поиск по конкретному слову** — находит все вхождения слова во всех документах с контекстами
2. **Топ-N униграмм по документу** — показывает самые частотные слова в конкретном документе

**Особенности:**
- Работает с лемматизированными текстами
- Фильтрует стоп-слова (предлоги, союзы, местоимения и т.д.)
- Показывает все контексты для каждого слова
- Проверяет соответствие количества контекстов частоте

```bash
python unigrams_search.py
```

**Пример вывода:**
```
📊 ТОП-20 УНИГРАММ В ДОКУМЕНТЕ ID: 15

 1. Лемма: "институт"
    Частота: 17 раз(а)
    Контексты:
      1. ...Ижевский механический ***институт*** начинался совсем не на улице...
      2. ...переданы в ведение Уральского политехнического ***института***...
```

---

### 2. Биграммы и коллокации (`bigrams_search.py`)

Поиск пар слов по частоте встречаемости и извлечение статистически значимых коллокаций.

**Режимы работы:**
1. **Поиск биграмм по слову** — находит все биграммы, содержащие заданное слово
2. **Топ-N биграмм по частоте** — показывает самые частотные пары слов в документе
3. **Биграммы во всех документах** — показывает топ-биграммы для каждого документа
4. **Коллокации по Likelihood Ratio** — извлекает статистически значимые пары слов

**Метрики:**
- **Частота (Raw Frequency)** — сколько раз биграмма встречается в тексте
- **Likelihood Ratio** — статистическая мера, показывающая значимость сочетания слов

```bash
python bigrams_search.py
```

**Пример вывода:**
```
📊 ТОП-15 КОЛЛОКАЦИЙ ПО LIKELIHOOD RATIO:

 1. Коллокация: "живой звук"
    Частота: 4 раз(а)
    Likelihood Ratio: 46.2161
    ВСЕ КОНТЕКСТЫ:
      1. ...состояться фестиваль ***живой звук*** барда галевуд...
      2. ...также участник клуб ижгту ***живой звук*** лауреат прошлый год...
```

---

### 3. Триграммы (`trigrams_search.py`)

Поиск троек слов по частоте встречаемости.

**Режимы работы:**
1. **Поиск триграмм по слову** — находит все триграммы, содержащие заданное слово
2. **Топ-N триграмм по частоте** — показывает самые частотные тройки слов в документе
3. **Триграммы во всех документах** — показывает топ-триграммы для каждого документа

```bash
python trigrams_search.py
```

---

### 4. Ключевые слова по TF*ICTF (`rake_search.py`)

Извлечение ключевых слов с использованием метрики TF*ICTF (Term Frequency — Inverse Collection Term Frequency) на основе коллекции документов одного раздела.

**Особенности:**
- Анализирует документы по разделам (chapters)
- Учитывает частоту слова в документе и редкость в коллекции
- Использует лемматизированные тексты
- Показывает контексты для каждого ключевого слова

```bash
python rake_search.py
```

**Процесс работы:**
1. Выберите раздел из списка
2. Выберите документ из раздела
3. Введите количество ключевых слов для вывода

---

### 5. Поиск по регулярным выражениям (`maska.py`)

Гибкий поиск по текстам с использованием регулярных выражений.

**Режимы работы:**
1. **Поиск во всех документах** — находит совпадения во всех документах
2. **Поиск в конкретном документе** — детальный поиск в выбранном документе
3. **Примеры и пояснения** — показывает примеры регулярных выражений

**Настройки поиска:**
- Учет регистра (включен/выключен)
- Тип текста (оригинальный/лемматизированный)
- Размер контекста

```bash
python maska.py
```

**Примеры регулярных выражений:**
- Имена: `[А-Я][а-я]+\s+[А-Я][а-я]+`
- Годы: `\d{4}\s+год`
- Телефоны: `8[\- ]?\d{3}[\- ]?\d{3}[\- ]?\d{2}[\- ]?\d{2}`

---

### 6. Извлечение именованных сущностей (`imen.py`)

Извлечение именованных сущностей из текстов: людей, организаций, мест, дат, денег и процентов.

**Режимы работы:**
1. **Извлечение из конкретного документа** — показывает все сущности в документе
2. **Статистика по всем документам** — общая статистика сущностей

**Типы сущностей:**
- 👤 Люди (PERSON)
- 🏢 Организации (ORGANIZATION)
- 📍 Места (LOCATION)
- 📅 Даты (DATE)
- 💰 Деньги (MONEY)
- 📊 Проценты (PERCENT)

```bash
python imen.py
```

## 🧠 Особенности обработки

### Лемматизация
- Используется библиотека `pymorphy3`
- Замена аббревиатур и сокращений (ИжГТУ → ижгту, ОМД → обработка_металлов_давлением)
- Сохранение защищенных слов (например, "ижгту")
- Подсчет уникальных лемм и общего количества токенов

### Стоп-слова
Модуль использует расширенный список стоп-слов, включающий:
- Все предлоги и союзы
- Частицы и междометия
- Местоимения
- Глаголы-связки
- Слова-паразиты
- Короткие слова (длиной ≤ 2 символа)

### Контексты
Для каждой найденной сущности/слова показываются:
- Все контексты с подсветкой искомого фрагмента
- Количество контекстов, соответствующее частоте
- Проверка соответствия количества контекстов частоте

## 📊 Схема данных

Модуль оперирует следующими таблицами:

```
documents (id, title, type, content, chapter_id, folder_id)
    └── documents_lemmatized (id, id_documents, content, unique_lemmas_count, total_tokens_count)
    └── chapters (id, name)
```

**Связи:**
- `documents.id` → `documents_lemmatized.id_documents`
- `documents.chapter_id` → `chapters.id`

## ⚙️ Конфигурация

### `db_connection.py`

| Переменная | Назначение |
|------------|------------|
| `DB_CONFIG` | Параметры подключения к PostgreSQL |
| `TABLES` | Словарь с названиями таблиц и их описаниями |

### Функции подключения
- `connect_db()` — создает соединение с БД
- `get_db_connection()` — создает соединение с DictCursor

### Вспомогательные функции
- `get_documents_by_chapter(chapter_id)` — получает документы раздела
- `get_all_chapters()` — получает все разделы
- `get_document_content(doc_id)` — получает содержимое документа
- `get_lemmatized_text(doc_id)` — получает лемматизированный текст
- `execute_query(query, params)` — выполняет SQL-запрос
- `get_table_count(table_name)` — получает количество записей в таблице

## 🐛 Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| `No module named 'pymorphy3'` | Установите: `pip install pymorphy3 pymorphy3-dicts` |
| `No module named 'nltk'` | Установите: `pip install nltk` |
| Ошибка подключения к БД | Проверьте `DB_CONFIG` в `db_connection.py` |
| Нет лемматизированных текстов | Запустите `documents_lemmatized.py` |
| Слишком много стоп-слов в результатах | Настройте `EXTRA_STOP_WORDS` в `unigrams_search.py` |

---

## 📝 Примечания по использованию

1. **Перед первым использованием** обязательно запустите `documents_lemmatized.py`
2. **Для работы с разделами** в `rake_search.py` убедитесь, что у документов заполнено поле `chapter_id`
3. **Регулярные выражения** чувствительны к регистру, используйте настройки поиска
4. **Все контексты** выводятся полностью, без ограничений по количеству
5. **Количество контекстов** всегда соответствует частоте вхождения
